## 三、定價策略（校正表）

### 你的定價 vs 市場數據

| 你的方案 | 月費 | 對標誰 | 市場驗證 | 支撐等級 |

| Pro | $199 | Mangomint $165、Mindbody $159 | 需充足 AI 價值支撐 | ⚠️ 需驗證 |

### 定價的未驗證假設

```
⚠️ CPC 不等於客戶意願
   CPC $9-11（美業關鍵字）表示廣告主願意付，不是你的客戶願意付
   必須問 10 個老闆：「這個工具值多少錢？」

⚠️ 一人美甲師月收入假設
   假設：$3,000-5,000/月
   軟體預算 1-3% = $30-150/月
   $29 Solo 在範圍內 ✅
   $99 Team 在上限邊緣 ⚠️
   必須用一手數據驗證

⚠️ 「免費轉付費」的轉換率假設
   假設：50-70% 轉換率
   業界 SaaS 免費試用轉換率：通常 2-5%
   但你是面對面銷售（不是線上自助），轉換率應更高
   目標 30-50% 比較現實
```

### 3 個 Kill Switch

```
Kill Switch 1（Month 3）：
  IF 付費客戶 < 5
  THEN 停下來找出：是產品問題？是銷售問題？是市場問題？

Kill Switch 2（Month 6）：
  IF 付費客戶 < 20
  THEN 重大調整：可能需要 pivot 垂直、改定價、找合夥人

Kill Switch 3（Month 9）：
  IF 付費客戶 < 50 AND 月流失 > 10%
  THEN 認真考慮：繼續投入是否值得
```

## 六、開發優先級矩陣（每週校正）

### Phase 1：MVP（Week 1-4）— 只做能讓第一個客戶付錢的東西

```
Week 1 開發前，先完成：
├── ❌ 未驗證假設 #1-6 必須在 Richmond 驗證
├── 如果驗證結果跟假設不同 → 調整 MVP 範圍
└── 不要在驗證前寫一行 code

Week 1-2（驗證後）：
├── DB: members, credit_packages, member_credits, services, staff, bookings
├── API: 會員 CRUD + SMS 發連結 + 預約 + 核銷
├── 客人端: /m/[token] 會員首頁 + 預約流程 + 驗證碼頁
├── 商家端: 今日列表 + 核銷頁面 + 會員管理
└── 複用: 已有的 AI 評論系統 + 通知系統

Week 3-4：
├── 離店 SMS 評論邀請（複用現有 QR 流程）
├── 預約前 24hr SMS 提醒（1 個 cron job）
├── 5 家店上線 + 每日追蹤
└── 收集 feedback → 決定 Phase 2 優先級
```

### Phase 2（Week 5-8）— 根據 feedback 決定做什麼

```
可能做的（按 feedback 排序，不是預設全做）：
├── 候補系統（如果老闆說「常有取消」）
├── 排班表（如果老闆說「排班很煩」）
├── 照片上傳（如果老闆說「想展示作品」）
├── 積分系統（如果客人對評論送積分有反應）
└── 沉睡喚醒（如果數據顯示回訪率需要提升）

不做：
├── AI 客服（太早，先手動處理）
├── AI 行銷助理（太早）
├── 跨行業模板（先把美甲做透）
└── 任何客人沒要求的功能
```

### Phase 3-4 根據 Phase 1-2 的數據決定，現在不規劃細節
